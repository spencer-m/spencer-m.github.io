Your Life Counter

Project Files


An Android Application developed for the Software Engineering Course

First of all, we have to choose what application we would like to do which is tied to a customer group. Then, we would ask them what they want the app to do, which is our requirements gathering. From here, we have also decided to do the agile methodology for the project. We have chosen this because of the flexibility it provides in our development process. The customer involvement is also a desirable feature of the methodology as it gives us feedback on the look, feel and capabilites of the app. After our requirements gathering, our customers have given us five functional requirements for the app. These are:
  • The calculation of calories required to be taken by the user
  • Keeping track of the user's calorie intake
  • Keeping track of the user's Water Intake
  • A database with the calorie count of common foods
  • An end of day comparison for the calories consumed
These requirements, with the exception of the last one, were then implemented to the app. In order to do this, we first started on doing a story map, low fidelity protptyping, storyboarding and use case scenarios to base our app onto. Our planning consisted of Poker Effort Estimation method so that we could distribute the workload of the project to the group member who can do it efficiently. We did our test using the Test-Driven Development Approach. And finally, in order to document our app, we used Structural and Behavioral modelling. These include UML diagrams the represent our app and sequence diagrams that describe important use case scenarios. GitLab was the platform used for version control. After all of these, we then presented our android application to the semester-end presentation session where other groups also presented. It was a fun project to do because I get to learn how the software development process works.

SnakeTRON

Project Files


A game developed as an assignment for the Computing Machinery II course
We used ARM Assembly to create this game for the Raspberry Pi Model B. The JTAG interface was used for flashing and debugging the program. A driver has to be made for the SNES controller the Raspberry Pi is interfaced with. On the course of the game development, we learned how to use the GPIO pins of the Raspberry Pi, how the interrupt works and how hardware and software "talk" to each other at the low level. We also learned more about source control along the course of the project. In the end, the resulting program is a TRON inspired Snake game which looks and plays really good.

Medical Clinic Database Management System

Project Files


The project is created for the Database Management Systems course
The project was announced early in the semester. We started with a proposal for it. Then an ER diagram was made to represent the database the user would manage. This ER diagram had undergone many revisions. There were also Hierarchical Input Process Output (HIPO) model and Data Flow Diagrams (DPD) that we made. Now that everything was ready, we went ahead and started coding the Database Management System (DBMS). Our languages of choice are mySQL, PHP and JavaScript. We used mySQL Workbench, WAMP/MAMP and BitBucket as our tools to make the project. I focused on the parts of the project where PHP and mySQL are involved. At the conclusion of the project, we have a Medical Clinic DBMS that implements these features:
  • Login System for Doctors, Employees and Patients
  • Patient Calendar
  • Patient Scheduler
  • Doctor Appointment Managementv
  • Inventory Management
  • Supplies Managementv
  • Billing Management
  • Drug Management

Hackathon 2016

Project Files


A Hitachi sponsored hackathon that lasted for two days
The hackathon's theme was "First World Problems". Late Day 1 was when we thought about which first world problem we should tackle. After an hour of brainstorming and discussion, we settled on the problem of many gamers: "How do I play with players that has a similar play style and skill level as me?" This is the basis of our app that we call the Friend Finder App. Now that we know where we are heading, we set out to learn developing apps for the Android platform. This is our first hackathon and we have not made a single Android app before. We also tried doing version control such as git but we moved on to do just doing file transfers because we could not get git working and we were running out of time. Our Day 1 ended with a working skeleton for our app. The next day we decided to start early. We kept learning about new things about the Android platform. As I worked on the framework and the interface, my other groupmates are working on the data structure, test database and visuals. At the end of the day, the app was 75% complete. Finally, on early Day 3, we finished up our app by improving the visuals, polishing the rough edges and preparing for our presentation.
